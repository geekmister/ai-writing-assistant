# v1.0.0 迭代（MVP）
## 一、产品规划
**版本**：1.0  
**目标发布形式**：GitHub 仓库，用户 clone 后本地运行（`npm install && npm run dev`）  

### 1. 产品定位
解决通用 AI 对话式写作的两大痛点：  

+ 生成后无法对已生成内容做段落重写、句子微调。  
+ 每次写作需要重复输入大量提示词来消除“AI 味道”或定义风格。

通过 **文档编辑器 + 选区操作 + 素材库 + 风格库** 的方式，让写作过程可迭代、风格可沉淀。

### 2. 目标用户
+ 网络小说作者  
+ 短视频文案写手  
+ 营销/公众号小编

### 3. 核心功能（全部必须实现）
#### 3.1 富文本编辑器（基于 TipTap + Vue 3）
+ 支持常见富文本格式：标题、加粗、斜体、列表、引用等。  
+ 支持选中任意段落、句子或自定义选区。  
+ 选区上方自动弹出 **悬浮工具栏**（常用操作）。  
+ 编辑器右侧提供 **右侧面板**（高级选项）。

#### 3.2 AI 重写与微调（核心操作）
对选中的文本，支持以下动作：  

##### 3.2.1 重写  
+ 完全改写选中内容，保持原意，改变表达方式。

##### 3.2.2 句子微调（全部支持）  
+ 缩写/缩短  
+ 扩写/丰富细节  
+ 改变语气（包含：正式、口语、幽默、严肃、温柔、霸道/强势、自卑/舔狗、高冷/绿茶）  
+ 改变人称（第一人称 ↔ 第三人称）  
+ 改变句式（长句拆短句、短句合并）

##### 3.2.3 生成结果呈现  
+ AI 一次性生成 **2～3 个不同版本**。  
+ 显示方式：左侧原文，右侧以标签页形式展示多个版本（版本1/版本2/版本3），点击标签切换。  
+ 每个版本下方有 **替换** 按钮，点击后覆盖原文。  
+ 支持 **重新生成** 按钮，再次请求 AI。

#### 3.3 素材库（剧情/金句/梗库）
##### 3.3.1 素材条目结构  
每条素材包含：  
+ 名称  
+ 内容（支持 **占位符 + 简单条件逻辑**，例如 `{{角色}}` 或 `如果心情=好 则 … 否则 …`）  
+ 手动标记条件（支持两种方式，用户任选其一或同时使用）：  
    - 自然语言描述（如“当用户写舔狗在深夜给女神发消息时”）  
    - 结构化标签（例如：角色类型、场景、情绪等键值对）
+ 向量嵌入字段（**预留**，MVP 阶段不实际生成向量，但字段存在）

##### 3.3.2 素材的调用方式  
+ **手动应用**：用户在编辑器选中文本后，从右侧面板的素材库中点选某个素材，AI 在生成时将该素材作为上下文注入。  
+ **自动检索（混合模式）**：  
    - 系统内部对当前写作上下文（段落、前后文、角色、场景）进行关键词匹配（MVP 使用简单关键词匹配）。  
    - 匹配到符合条件的素材时，自动将其注入 AI 提示词。  
    - 用户可在设置中开启/关闭自动检索。

##### 3.3.3 素材库管理  
+ 支持创建、编辑、删除素材。  
+ 素材存储在浏览器 IndexedDB 中，可导出/导入 JSON。

#### 3.4 个人写作风格库
##### 3.4.1 风格创建方式（四种全支持）  
+ **手动填写描述**：用户输入一段自然语言风格描述（如“短句、第一人称、自嘲口吻”）。  
+ **上传范文自动提取**：  
    - 第一步：前端本地 NLP 统计（句长、词性、人称代词频率等），生成基础标签（如“中短句、第一人称、少用形容词”）。  
    - 第二步：用户可选“调用 AI 深化分析”，将范文和本地标签发送给 OpenAI，返回更细腻的风格描述。
+ **自动沿用**：  
    - 全局默认风格（设置中指定）。  
    - 风格选择器（下拉框，写作时手动切换）。  
    - 智能推荐（基于当前文档内容推荐风格，用户确认后采用）。
+ **手动微调**：对任何自动生成的风格描述，用户可自由编辑修改。

##### 3.4.2 风格的应用  
+ 所有 AI 生成（重写、微调）默认使用 **当前选中的风格**（若未选则用全局默认风格）。  
+ 风格可保存为多个预设，一键切换。

#### 3.5 AI 服务配置（开源版专有）
+ 用户在 **前端设置面板** 填写 OpenAI API Key。  
+ 支持选择模型：GPT-3.5 或 GPT-4。  
+ 所有 AI 请求直接从浏览器发往 OpenAI API（无需后端）。  
+ 前端需处理 API Key 安全（仅存在浏览器 localStorage，不发送到任何第三方服务器）。

## 二、技术方案
### 2.1 总体架构
**纯前端三层架构**：

+ **展示层**：Vue 3 + TipTap 编辑器 + Naive UI 组件
+ **逻辑层**：Pinia 状态管理 + 自定义服务（AI 调用、上下文构建、NLP 分析）
+ **存储层**：IndexedDB（Dexie.js 封装） + localStorage（API Key 等简单配置）

**数据流**：

1. 用户操作 → Vue 组件事件 → Pinia action
2. Action 调用服务（如 `openai.ts`）→ 构建提示词 → fetch OpenAI API
3. 返回结果 → 更新 Store → 视图更新
4. 同时持久化到 IndexedDB（防丢失）

#### 2.5.2 AI 调用模块
**openai.ts** 详细实现：

```typescript
// services/ai/openai.ts
import type { AppConfig } from '@/types/config';

export async function callOpenAI(
  messages: Array<{ role: 'system' | 'user' | 'assistant', content: string }>,
  config: AppConfig
): Promise<string[]> {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${config.openaiApiKey}`,
    },
    body: JSON.stringify({
      model: config.model,
      messages,
      temperature: config.generation.temperature,
      max_tokens: config.generation.maxTokens,
      n: config.generation.n,
      stream: false,
    }),
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.error?.message || 'OpenAI API 请求失败');
  }

  const data = await response.json();
  // data.choices 是一个数组，每个 choice 有 message.content
  const contents = data.choices.map((choice: any) => choice.message.content.trim());
  return contents;
}
```

**上下文构建器**（`contextBuilder.ts`）：

```typescript
import { useMaterialStore } from '@/stores/materialStore';
import { useStyleStore } from '@/stores/styleStore';
import { REWRITE_PROMPT } from './prompts';

export async function buildRewritePrompt(
  selectedText: string,
  beforeText: string,
  afterText: string,
  instruction: string,
  styleId: string | null,
  autoRetrieveMaterial: boolean
): Promise<string> {
  const styleStore = useStyleStore();
  const materialStore = useMaterialStore();

  let styleDesc = '';
  if (styleId) {
    const style = styleStore.getStyleById(styleId);
    styleDesc = style?.description || '';
  } else {
    const defaultStyle = styleStore.getDefaultStyle();
    styleDesc = defaultStyle?.description || '无特定风格';
  }

  let materialsText = '';
  if (autoRetrieveMaterial) {
    const contextText = `${beforeText}\n${selectedText}\n${afterText}`;
    const matched = materialStore.retrieveByKeywords(contextText);
    if (matched.length > 0) {
      materialsText = matched.map(m => `- 【${m.name}】${m.content}`).join('\n');
    }
  }

  const prompt = REWRITE_PROMPT
    .replace('{{style}}', styleDesc)
    .replace('{{materials}}', materialsText || '无')
    .replace('{{beforeText}}', beforeText)
    .replace('{{selectedText}}', selectedText)
    .replace('{{afterText}}', afterText)
    .replace('{{instruction}}', instruction);

  return prompt;
}
```

**MaterialStore 核心方法**：

```typescript
// stores/materialStore.ts
import { defineStore } from 'pinia';
import { db } from '@/services/storage/db';
import type { Material } from '@/types/material';

export const useMaterialStore = defineStore('material', {
  state: () => ({
    materials: [] as Material[],
  }),
  actions: {
    async loadMaterials() {
      this.materials = await db.materials.toArray();
    },
    async addMaterial(material: Omit<Material, 'id' | 'createdAt'>) {
      const newMaterial: Material = {
        id: uuid(),
        ...material,
        createdAt: Date.now(),
      };
      await db.materials.add(newMaterial);
      await this.loadMaterials();
    },
    retrieveByKeywords(text: string): Material[] {
      const lowerText = text.toLowerCase();
      return this.materials.filter(m => {
        if (m.naturalCondition && lowerText.includes(m.naturalCondition.toLowerCase())) {
          return true;
        }
        if (m.structuredTags) {
          const tagValues = Object.values(m.structuredTags).flat();
          return tagValues.some(tag => lowerText.includes(tag.toLowerCase()));
        }
        return false;
      });
    },
    // 其他 CRUD 方法...
  },
});
```

#### 2.5.4 风格库模块
**本地 NLP 统计**（`styleAnalyzer.ts`）：

```typescript
import nlp from 'compromise';

export function analyzeLocalStyle(text: string) {
  const doc = nlp(text);
  const sentences = doc.sentences().out('array');
  const avgLength = sentences.reduce((sum, s) => sum + s.length, 0) / sentences.length;
  const firstPersonCount = doc.match('(i|me|my|mine|we|us|our)').length;
  const totalWords = doc.terms().out('array').length;
  const firstPersonRatio = totalWords > 0 ? firstPersonCount / totalWords : 0;

  // 简单语气判断
  const informalWords = ['gonna', 'wanna', 'kinda', 'like', 'umm', 'yeah'];
  const formalWords = ['however', 'therefore', 'consequently', 'accordingly', 'nevertheless'];
  const informalCount = doc.match(informalWords.join('|')).length;
  const formalCount = doc.match(formalWords.join('|')).length;
  const tone = informalCount > formalCount ? 'informal' : 'formal';

  return {
    avgSentenceLength: avgLength,
    firstPersonRatio,
    tone,
    wordCount: totalWords,
  };
}
```

**GenerationStore**：

```typescript
// stores/generationStore.ts
export const useGenerationStore = defineStore('generation', {
  state: () => ({
    loading: false,
    versions: [] as string[],
    selectedVersionIndex: 0,
  }),
  actions: {
    async generate(request: GenerationRequest) {
      this.loading = true;
      try {
        const prompt = await buildRewritePrompt(
          request.selectedText,
          request.beforeText,
          request.afterText,
          request.instruction,
          request.styleId,
          request.autoRetrieveMaterial
        );
        const messages = [{ role: 'user', content: prompt }];
        const config = useConfigStore().config;
        const rawVersions = await callOpenAI(messages, config);
        // callOpenAI 已经返回 string[]
        this.versions = rawVersions;
        this.selectedVersionIndex = 0;
      } finally {
        this.loading = false;
      }
    },
    replaceSelectedVersion(editor: Editor) {
      const version = this.versions[this.selectedVersionIndex];
      if (version && editor) {
        editor.commands.insertContent(version);
        // 关闭面板等后续操作
      }
    },
  },
});
```

### 2.7 性能优化与安全
+ **API Key 安全**：仅存储在 localStorage，不发送到任何后端；用户自行承担 Key 安全责任。
+ **XSS 防护**：TipTap 输出的 HTML 在保存前和渲染时使用 `DOMPurify.sanitize`。
+ **并发控制**：每次 AI 请求前检查 `generationStore.loading`，若为 true 则禁用按钮；使用 `AbortController` 支持取消。
+ **本地存储压缩**：IndexedDB 存储 HTML 字符串，无额外压缩，但现代浏览器足够；可选后续加入 LZ-string。

### 4.3 常见问题
**Q：用户 API Key 安全吗？**  
A：Key 只存在浏览器 localStorage，所有请求从用户浏览器直接发往 OpenAI，不经过任何中间服务器。但无法防止用户自己泄露 Key，建议用户使用限额 Key。

**Q：支持其他 AI 服务吗？**  
A：MVP 仅支持 OpenAI，后续开源版可扩展通过配置 `apiBase` 支持兼容接口的服务（如 Azure OpenAI、本地 LM Studio）。

**Q：多文档管理？**  
A：MVP 支持创建、保存、删除多个文档，文档列表在侧边栏（简单实现）。

**Q：如何备份数据？**  
A：通过“导出”功能下载 JSON 文件，需要时“导入”恢复。
